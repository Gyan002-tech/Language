# Teaching Notes

## Preferences
- Wants **multiple concrete worked examples per real-world context** (meeting, conversation, debate,
  storytelling) before moving to a new technique — one abstract example is not enough to feel like he can
  apply a framework across settings. Err on the side of more worked examples per context, even if it means
  slowing down before introducing the next technique.
- Course plan (subject to change as learning records accumulate):
  1. **Structure** — organizing a spontaneous answer (PREP framework) → applied across meetings, conversation,
     debate, storytelling with multiple worked examples each. Lessons 0001-0003 done; meeting phrase-bank
     scaffold confirmed effective (see learning-records/0003).
  2. **Word selection confidence** *(reprioritized ahead of module 3/4, per user's own stated priority)* —
     not retrieval speed but *doubt*: grammatical/collocation fit, "is there a better word", and "do natives
     actually say this." Techniques: approximation/good-enough-for-now heuristic (Dörnyei & Scott 1997),
     collocation pattern awareness, YouGlish/Ludwig as an after-the-fact confidence-building tool (not
     mid-conversation).
  3. **Pronunciation** *(reprioritized alongside module 2)* — user doesn't know which specific sounds are
     the problem (general self-consciousness, no concrete symptom). No audio channel available to me, so
     this must start as a **self-diagnostic** lesson: the "Comma Gets a Cure" IDEA passage (broad phonetic
     coverage) + self-record-and-compare method, before any targeted sound drilling.
  4. **Fluency & delivery** — shadowing practice, filler-word replacement, smoothing out pauses.
  5. **Integration** — combining all of the above in real exchanges; graduating to live practice
     (Toastmasters / iTalki) for wisdom, not just solo drills.
